# 洋書翻訳システム 要件定義書

## 1. プロダクト概要

英語書籍のPDFをアップロードすると、章ごとにサマリ付きで日本語訳PDFを生成するWebアプリ。

**販売モデル**：B2B（企業向け）
**価格**：初期 10万円 + 月額 5,000円
**ターゲット**：英語書籍を業務で扱う企業（コンサル、研修会社、研究機関など）

## 2. 技術スタック

| 項目 | 採用技術 |
|------|---------|
| 言語 | Python 3.11+ |
| Webフレームワーク | Streamlit |
| ホスティング | Streamlit Community Cloud（無料、GitHubと連携） |
| PDF抽出 | PyMuPDF (fitz) |
| PDF生成 | ReportLab + 日本語フォント (Noto Sans JP) |
| 状態管理 | SQLite（ジョブ履歴）+ Streamlit Session State |
| LLM | Gemini / OpenAI / Anthropic Claude（顧客選択） |

## 3. 機能要件

### 3.1 顧客向け機能

#### ログイン
- メールアドレス＋パスワードで認証
- 1社につき複数ユーザー登録可能
- 管理者ユーザーが他ユーザーを追加できる

#### 設定画面
- **使用するLLMプロバイダー選択**（Gemini / OpenAI / Claude）
- **APIキー登録**（暗号化保存）
- **翻訳スタイル**（ですます調 / である調）
- **章サマリの長さ**（短め150字 / 標準250字 / 長め400字）

#### 翻訳実行
1. PDFファイルをドラッグ＆ドロップ
2. ボタン1つで翻訳開始
3. 進捗バー表示（章ごとの進捗）
4. 完了後、日本語訳PDFをダウンロード

#### 履歴
- 過去の翻訳ジョブ一覧
- ステータス（処理中・完了・失敗）
- 元PDFと訳PDFを再ダウンロード可能（30日間保持）

### 3.2 章検出（コア機能）

**LLMで章構造を解析する方式**（正規表現は使わない）：

1. PDFの最初の10ページからテキスト抽出
2. LLM（顧客選択のプロバイダー）に「目次を読んで章一覧を返して」と依頼
3. 返ってきた章タイトル一覧を全文中で検索→位置特定
4. 各章のテキスト範囲を確定

**フォールバック**：
- 目次から章リストが取れない場合 → LLMに「本文を読んで論理的なセクションに分けて」と依頼
- それでも失敗時 → 10章程度の文字数均等分割

### 3.3 翻訳処理

- 各章を1つずつLLMに渡して翻訳＋サマリ生成
- 長い章は内部でチャンク分割（6,000文字程度ずつ）
- 並列処理で高速化（5並列程度）
- 失敗時は3回まで自動リトライ

### 3.4 出力PDF仕様

- 表紙（書名・翻訳日・章数）
- 各章冒頭にサマリ（指定文字数）
- 本文（指定スタイル）
- 章ごとに改ページ
- 日本語フォント埋め込み

### 3.5 管理者機能（社内向け）

- 全顧客アカウント一覧
- 月次利用状況（ジョブ数・処理時間）
- 顧客追加・削除
- パスワードリセット

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| セキュリティ | APIキーは AES-256 で暗号化してDB保存 |
| パフォーマンス | 300ページの本を15〜30分以内に処理完了 |
| 可用性 | 平日9〜18時に動作すれば良い（Streamlit Cloud準拠） |
| データ保持 | 翻訳済みPDFは30日間サーバー保持、その後自動削除 |
| 法的事項 | プライバシーポリシー・利用規約をフッター表示 |

## 5. 課金モデル

**BYOK（Bring Your Own Key）**：
- LLM APIの利用料金は **顧客が直接プロバイダーに支払う**
- 当社（Maki社）はホスティング費＋アプリ使用権のみを月額で請求
- 当社のリスク：API代の負担なし、固定費（ホスティング）のみ

## 6. 対応PDF条件

| 項目 | 条件 |
|------|------|
| 種類 | OCR済み（テキスト埋め込み済み）PDFのみ |
| サイズ | 50MB以内 |
| ページ数 | 600ページ以内 |
| 暗号化 | パスワード保護されていないもの |

非対応PDFはアップロード時にエラー表示。

## 7. 顧客導入フロー（1〜2時間で完了）

| 時間 | 作業 | 担当 |
|------|------|------|
| 0-15分 | アカウント発行（管理画面で当社側で作成） | 当社 |
| 15-30分 | LLMプロバイダーの選定＋APIキー取得方法レクチャー | 当社 |
| 30-45分 | 顧客のAPIキー取得＋アプリへの登録 | 顧客 |
| 45-75分 | テストPDFで動作確認 | 顧客 |
| 75-90分 | 業務での使い方トレーニング | 当社 |

## 8. リリース後の運用

- LLMプロバイダーの仕様変更（年1〜2回）に追随
- ライブラリの脆弱性パッチ
- 月次バックアップ（顧客データ）

## 9. 契約書に明記する事項

- 顧客のAPI課金は顧客の責任
- APIキー漏洩時の損害賠償は月額3ヶ月分を上限
- 無償サポートはサブスク継続中のみ
